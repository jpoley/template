# Diagrams

Authoritative source for every diagram in this repo is a `.mmd` (Mermaid) file here. PNGs are generated by `scripts/build-diagrams.sh` and committed alongside the sources so readers with any Markdown viewer see the rendered image.

## Regenerate

```bash
./scripts/build-diagrams.sh              # rebuild all
./scripts/build-diagrams.sh foo.mmd      # rebuild one
```

Runs `minlag/mermaid-cli` in a throwaway Docker container — no host install needed. First run pulls the image (~100 MB); subsequent runs are offline.

## Why commit both `.mmd` and `.png`?

- The `.mmd` is the truth and gets diffed in PRs — humans can read it.
- The `.png` means the diagram shows up in any Markdown renderer, including offline ones, without requiring a Mermaid plugin.
- GitHub also renders \`\`\`mermaid fenced code blocks natively, so where that's available (e.g. in an issue or wiki) you can paste the raw source.
